# Accessibility Compliance

## Standards
- WCAG 2.2 Level AA targeting

## Practices Implemented
- Color contrast meets AA for text and interactive elements
- `aria-haspopup`, `aria-expanded`, and `aria-controls` on dropdown triggers
- Form inputs labeled and keyboard accessible
- Focus-visible styles for buttons and links
- Semantic roles for menus and alerts

## Testing Checklist
- Keyboard-only navigation flows
- Screen reader labels for inputs and action buttons
- Skip links and landmark regions (header/main)
- Reduced motion respects user preferences
